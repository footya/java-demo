## 输出与交付

- 输出语言：中文。
- 默认只输出必要内容：除非用户明确要求，不要写 README/文档说明/使用说明/总结/示例代码/测试代码。
- 必须保证可执行：修改后项目应能通过 `mvn -q -DskipTests package`。
- 变更边界：只改为完成需求所必需的最小范围；不要顺手重构/格式化全项目。

## 中文注释（教学型，偏多）

- 在新增/修改的 Java 代码中，尽可能补充中文注释，重点覆盖：
  - 类/方法职责与关键设计意图
  - 关键业务分支与边界条件
  - 入参/出参字段含义（尤其是接口 DTO）
  - 异常原因与处理策略
- 避免低价值注释（例如逐行复述语句做什么）。

## 可维护性与准确性

- 优先选择简单、可读、可验证的实现；不要引入不必要的新框架/新模式。
- 日志使用 SLF4J 参数化；不要使用 `System.out.println`。
- 不要泄露敏感信息（token、key、身份证号、手机号等）到日志/返回值中；必要时脱敏。
