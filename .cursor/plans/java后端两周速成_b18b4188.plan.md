# 两周每天1小时：Java后端（Spring Boot）速成计划

## 目标与验收

- **目标**：能独立完成一个包含 REST API、参数校验、统一异常、数据库 CRUD、日志、认证、部署的 Spring Boot 小服务。
- **验收**：你能在 30–60 分钟内从 0 新增一个业务接口（含 DTO/校验/Service/Repository/DB 表变更/错误码/接口文档），并能本地运行与打包。

## 学习路径（只做最核心闭环）

- **语言与工程基础**：Java 基础语法 + Maven + IDE 调试
- **框架主线**：Spring MVC（Controller/DTO/Validation）→ Service 分层 → 数据访问（JPA 或 MyBatis 二选一）
- **工程化必备**：统一异常/错误码、日志、配置管理、接口规范
- **上线意识**：打包、配置、健康检查、可观测性（最小）

## 每日安排（14 天，每天 60 分钟）

> 每天结构固定：10 分钟概念 + 40 分钟动手 + 10 分钟复盘（记录 3 条要点和 1 个问题）。

### 第 1–3 天：跑通 Spring Boot 与 REST

- **Day1**：环境与工程
- 产出：本地能跑 `Spring Boot`，会用断点调试、看日志、看启动配置。
- **Day2**：REST 基础
- 产出：掌握 `Controller`、路由、请求/响应体、状态码；能写 2 个简单接口。
- **Day3**：DTO 与参数校验
- 产出：引入 `Validation`，完成请求 DTO、校验失败返回。

### 第 4–6 天：分层与错误处理

- **Day4**：Service 分层与依赖注入
- 产出：Controller 只做入参/出参，业务进 Service。
- **Day5**：统一异常与错误码
- 产出：全局异常处理（如 `@ControllerAdvice`），标准化错误响应结构。
- **Day6**：日志与可观测最小集
- 产出：关键链路日志（入参/耗时/错误），基础运行信息可定位问题。

### 第 7–10 天：数据库 CRUD（建议 JPA，追求快）

- **Day7**：选型与连库
- 产出：确定 **JPA 或 MyBatis**；本地连接数据库（建议先用 H2/本地 MySQL）。
- **Day8**：建表与实体映射
- 产出：1 张核心表 + 实体/Mapper/Repository。
- **Day9**：CRUD 接口
- 产出：新增/查询/更新/删除 4 类接口各 1 个，返回结构统一。
- **Day10**：分页、排序、条件查询
- 产出：列表接口支持分页参数与基础条件过滤。

### 第 11–12 天：认证与接口规范

- **Day11**：最小认证
- 产出：实现一种最小可用方案：
- 选项 A：API Key（简单快速）
- 选项 B：JWT（更通用）
- **Day12**：接口规范化
- 产出：统一响应格式、错误码表、接口命名与版本策略（如 `/api/v1`）。

### 第 13–14 天：打包部署与复盘

- **Day13**：打包与配置
- 产出：`mvn package` 产物可运行；区分 dev/prod 配置。
- **Day14**：端到端演练
- 产出：从需求到落地：新增一个小功能（带 DB 变更 + 校验 + 错误码 + 认证），并做一次完整自测与复盘清单。

## 你现有仓库的建议切入点

- 以现有 Spring Boot 工程为基础，优先把：
- **统一异常/响应**
- **DTO 校验**
- **数据库 CRUD**
这三块补齐，再考虑认证与部署。
